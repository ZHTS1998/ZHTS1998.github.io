##1.SDS字符串：

1.1sds字符串定义：

    struct sdshdr{
        int len;
        int free;
        int buf[]; 
    }

其中SDS->buf也以’\0’结尾，这样使得SDS可以复用C的函数

len为字符串的长度

1.2SDS与C的比较

获得长度O（1）

SDS由于有len属性，因此获取字符串长度这一操作为O(1),而C语言需要遍历到'\0'
由于REDIS中可能有很多时候需要读取字符串长度，因此这一优化使得获取字符串长度的工作不会成为Redis的性能瓶颈

不担心缓冲区溢出

SDS具有自动扩容的机制，每次对字符串修改前会检查SDS空间大小，若不够的话会对其进行扩容

减少修改字符串时带来的内存重分配次数

redis对速度要求苛刻，而c语言的字符串修改时会进行内存重分配，否则会溢出或泄露；SDS字符串可以具有未使用空间，用
free来记录，这样来减少内存重分配次数

1.3SDS空间分配：

1.3.1预分配规则如下：

简而言之，小于1MB时，分配两倍+1个字节（’\0’），大于1MB时，多分配1MB+1个字节。

通过预分配，减少了重分配的次数，进而减少了和内存的交互，提高了效率。相当于空间换时间。

1.3.2惰性空间释放

伪释放，只是使用SDS->free将没用到的空间记录下来，并没有真正的减少大小。当然也有相应的API来真正释放这些内存。

1.4SDS的二进制安全：

当C语言遇到上面这幅图时，C语言在读到REDIS’\0’后就停止了，SDS则不会，由于SDS记录了它的len属性，因此会一直读取长度为
len的字符。

也就是说，Redis不是用这个数组来保存字符，而是用它来保存一系列二进制数据。程序不会对其中的数据做任何限制、过滤、或者假设，数据在写入时是什么样的，它被读取时就是什么样。

也正因此，SDS只能使用部分C语言中的字符串函数。而不是所有

SDS总结：SDS优点如下：

1）常数复杂度获取字符串长度。

2）杜绝缓冲区溢出。

3）减少修改字符串长度时所需的内存重分配次数。

4）二进制安全。

5）兼容部分C字符串函数。
